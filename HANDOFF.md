# YouTrade Bot — Саммари для разработчика

## Что это

Telegram-бот для YouTrade Prop. Два режима в одном боте:
- **Аружан** (sales): продаёт челлендж новым юзерам через Claude AI
- **Алия** (support): после покупки ведёт по 8 шагам онбординга

Плюс CRM для команды: смотреть диалоги, перехватывать на себя, вручную помечать купивших.

---

## Что надо сделать по шагам

### 1. Развернуть бота на сервере (CentOS 8/9)

Всё в `README.md` — там пошагово: установка пакетов, PostgreSQL, nginx, SSL, systemd.

**Нужные ключи (запросить у Романа):**
- `TELEGRAM_BOT_TOKEN` — из @BotFather
- `CLAUDE_API_KEY` — `console.anthropic.com` (уже оплачено $20)
- Домен `bot.youtrade.kz` — поднять поддомен с DNS на сервер

**Два systemd-сервиса:**
```bash
sudo cp configs/youtrade-bot.service /etc/systemd/system/
sudo cp configs/youtrade-worker.service /etc/systemd/system/
sudo systemctl enable --now youtrade-bot      # основной бот
sudo systemctl enable --now youtrade-worker   # фоновый воркер для ретеншна
```

### 2. Выдать Роману доступы

- URL админки: `https://bot.youtrade.kz/admin?key=<ADMIN_DASHBOARD_KEY>`
- URL CRM: `https://bot.youtrade.kz/crm?key=<ADMIN_DASHBOARD_KEY>`
- Telegram username бота

### 3. Проверить что работает

После деплоя зайти в Telegram к боту, написать `/start`:
1. Должно прийти 6 welcome-сообщений с фото (5 текстовых + 2 фото)
2. Аружан представится и спросит про опыт
3. Ответить что-то — должен прийти ответ от Аружан через Claude

Если сломалось — логи: `sudo journalctl -u youtrade-bot -f`

## Что НЕ работает автоматически (требует интеграции)

### 1. Автофиксация оплаты ⚠️

Сейчас бот узнаёт что юзер купил **только когда менеджер вручную нажал кнопку "Отметить купил"** в CRM.

Чтобы сделать автомат, нужно:
- Платёжка (CloudPayments / Stripe / USDT-processor) отправляет webhook после успешной оплаты
- В боте добавить endpoint `POST /purchase-webhook` который принимает уведомление и ставит `purchased=True`
- Передавать `uid` юзера в metadata платежа (чтобы знать кого отмечать)

**Что запросить у Романа:**
- Какая платёжная система используется
- Документация на их webhook-уведомления
- Доступ к настройкам чтобы прописать URL webhook

После этого я (Роман) допишу endpoint, разработчик просто задеплоит обновление.

### 2. Скриншоты MatchTrader для онбординга ⚠️

8 файлов в `/opt/youtrade-bot/screenshots/`:
- `01_login.jpg` — страница логина
- `02_main_screen.jpg` — главный экран после входа
- `03_buy_sell_buttons.jpg` — кнопки BUY/SELL с объёмом
- `04_open_positions.jpg` — список открытых позиций с P/L
- `05_stop_loss.jpg` — окно настройки стоп-лосса
- `06_close_position.jpg` — кнопка закрытия позиции
- `07_prop_tab.jpg` — вкладка Prop с правилами
- `08_completed.jpg` — успешное завершение (опционально)

Пока скринов нет — бот просто шлёт текст шагов без картинок. Скрины добавятся когда Роман соберёт.

### 3. Ссылки в ретеншн-пушах 📝

В `app/retention.py` есть URL-плейсхолдеры:
```python
"academy_url": "https://youtradeprop.com/academy",
"guide_prop_url": "https://youtradeprop.com/guides/what-is-prop",
"guide_risk_url": "https://youtradeprop.com/guides/risk-management",
```

Заменить на реальные когда Роман выложит `hub-ru.html`, `academy-ru.html` и гайды на продакшен.

### 4. Уведомления операторам при takeover 🔔

Сейчас CRM показывает входящие сообщения только при обновлении страницы. Если менеджер взял диалог на себя — он должен сам мониторить.

Варианты на будущее (пока не реализовано):
- Отдельный Telegram-бот для оператора который пингует его при новом сообщении
- Звук на странице CRM при новом сообщении через WebSocket/SSE

### 5. Ретеншн для других сегментов 📝

Пока работает только для "engaged non-purchaser" (поговорили 3+ сообщений, не купили).

Не реализовано (добавим позже):
- Купил но не начал онбординг
- Застрял на шаге
- Слил челлендж (max loss)
- Купил и прошёл онбординг но не торгует 7+ дней

### 6. Real-time данные по выплатам 📝

В `app/content.py` есть `PAYOUTS_WEEK` — там топ выплат. **Обновляется вручную** раз в неделю через редактирование файла и рестарт бота.

Автоматизация (когда будет готово):
- Endpoint `/admin/update-payouts?key=...` с формой чтобы менеджер сам обновлял через веб
- Или интеграция с системой учёта выплат (API)

### Технологии

- **Python 3.11** + venv
- **FastAPI** (uvicorn) — веб-сервер
- **python-telegram-bot 21.6** — работа с Telegram API
- **SQLAlchemy async** + asyncpg — PostgreSQL
- **anthropic** — Claude API (Haiku 4.5)
- **pydantic-settings** — конфиг из .env

### Два процесса в systemd

1. **youtrade-bot** — основной, слушает `/webhook` на 8000
2. **youtrade-worker** — фоновый, раз в 5 минут проверяет кого пушнуть

## Как обновлять код в будущем

Роман присылает обновление как tar.gz:

```bash
sudo bash update.sh /tmp/update.tar.gz
```

Скрипт сам: бэкапит БД, разворачивает код, ставит новые зависимости, применяет миграции, перезапускает бота.

Если менялась схема БД — миграция применится автоматически (через SQLAlchemy `create_all`, он добавляет новые колонки).

## Метрики (что смотреть в админке)

**`/admin?key=...`:**
- Всего юзеров
- Активных 24ч/7д
- Купили / Завершили онбординг / На руках у менеджеров
- **Воронка продаж**: Старт → Welcome показан → Вступил в диалог → Кликнул оплату → Купил
- **Воронка онбординга**: по 8 шагам для купивших
- **A/B группы**: сравнение A/B/C по всем метрикам
- Последние 30 вопросов от юзеров

**`/crm?key=...`:**
- Список диалогов с фильтрами (все / активные / купили / на руках / застрявшие)
- По клику — вся история переписки + кнопки управления
